# Prompts utilizados

## Prompt 1 - Contexto y alcance
"Actúa como Product Manager y Software Architect. Diseña la versión inicial de un ATS para una startup llamada LTI. Incluye propuesta de valor, ventajas competitivas y funcionalidades principales orientadas a eficiencia de HR, colaboración en tiempo real y automatización con IA."

## Prompt 2 - Lean Canvas
"Genera un Lean Canvas para LTI ATS con: problema, segmentos de clientes, propuesta única de valor, solución, canales, ingresos, costes, métricas clave y ventaja injusta. Entrega el resultado en formato de diagrama Mermaid."

## Prompt 3 - Casos de uso
"Define 3 casos de uso críticos para un ATS moderno: publicación de vacante, screening/shortlist asistido por IA y entrevistas colaborativas con decisión final. Para cada caso, incluye actores, objetivo, precondiciones, flujo principal, alternos y un diagrama Mermaid."

## Prompt 4 - Modelo de datos
"Propón un modelo de datos relacional para el ATS, con entidades, atributos (nombre y tipo), y relaciones. Incluye entidades clave como Company, User, JobOpening, Candidate, Application, Interview, Scorecard y Offer. Añade un diagrama ER en Mermaid."

## Prompt 5 - Arquitectura de alto nivel
"Diseña la arquitectura de alto nivel del sistema LTI ATS. Incluye frontend, gateway/BFF, servicios core, servicio de IA, automatizaciones, integraciones externas, persistencia y analítica. Entrega una explicación breve y un diagrama Mermaid."

## Prompt 6 - C4 en profundidad
"Crea un diagrama C4 para LTI ATS y profundiza en un componente a elección. Incluye al menos contexto, contenedores y componentes internos del servicio elegido, con foco en responsabilidades e interacciones."

## Prompt 7 - Ajuste final
"Consolida toda la documentación en un único markdown claro y profesional, en español, con secciones numeradas y diagramas Mermaid listos para ser versionados en GitHub."
